### Deploy Tracking (Phase 2)

The "Ready to Deploy" and "Monitoring" states are designed to support automatic deploy tracking:

**Ready to Deploy Detection** (Future):
- Detect when MR is merged to `main` branch
- Check if associated ticket is completed
- Auto-transition to "Ready to Deploy"

**Released to Staging Detection** (Future):
- Monitor ArgoCD deployments
- Detect when build reaches staging environment
- Sub-state within "Ready to Deploy"

**Monitoring Detection** (Future):
- Detect manual deploys to production
- Auto-transition to "Monitoring"
- Manual transition to "Done" when monitoring complete

## Phase 2 Roadmap

### JIRA Deploy Tracking
1. **MR Merge Detection**
   - Subscribe to GitLab webhooks for MR merge events
   - Parse commit messages for ticket keys
   - Check if ticket status allows "Ready to Deploy"

2. **Deploy Detection**
   - Monitor ArgoCD deployments
   - Correlate build IDs with commits
   - Extract JIRA ticket keys from commit history
   - Auto-transition tickets through states

3. **Manual Transitions**
   - Add ticket detail modal
   - Transition buttons (e.g., "Mark as Monitoring", "Mark as Done")
   - Update JIRA status via API

### WX Deployments Integration
1. **ArgoCD Integration**
   - Query application status API
   - Extract current image tags
   - Parse build IDs from deployment specs
   - Monitor sync status for health

2. **GitLab Integration**
   - Resolve commit URLs from SHAs
   - Fetch commit metadata
   - Show commit messages in tooltip

3. **Tigercli Integration**
   - Generate correct deploy URLs
   - Link to specific environment deploys
   - Show deploy history

4. **Real-time Updates**
   - WebSocket connection to ArgoCD
   - Live deploy notifications
   - Health status updates
